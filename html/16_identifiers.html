Identifiers
A Python identifier is a name used to identify a variable, function, class, module or other object.
An identifier starts with a letter A to Z or a to z or an underscore (_) followed by zero or more letters, underscores and digits (0 to 9)
Python does not allow punctuation characters such as - @, $, and % within identifiers.
Python is a case sensitive programming language.
Cannot start with number, 123tom is error, but tom123 OK
underscore OK
cannot have special characters such as $ sign OR a minus sign
_123tom OK
_tom Ok
alphabet symbols, upper or lowercase OK
_Ram =10 but PRIVATE
identifier cannot start with digit
cannot use reserved keyword as identifier // note
Best practice in Python is to use camelCase for variables (J.Manibog)
Syntax
Identifier Conventions
Class names start with an uppercase letter. All other identifiers start with a lowercase letter.
Starting an identifier with a single leading underscore indicates that the identifier is private.
Starting an identifier with two leading underscores indicates a strongly private identifier.
If the identifier also ends with two trailing underscores, the identifier is a language-defined special name.
Examples
str.isidentifier() 

    'tom'.isidentifier()     //    True
    'to-m'.isidentifier()    //    False
    '1tom'.isidentifier()    //    False
    'tom$'.isidentifier()    //    False
    '_tom'.isidentifier()    //    True

    import keyword
    keyword.iskeyword('def') //    True
    'def'.isidentifier()     //    True
Notes
Valid Examples
ab10c // contains only letters and numbers
abc_DE // contains all the valid characters
_ // surprisingly but Yes, underscore is a valid identifier
_abc // identifier can start with an underscore
myfirst_name = "John"
Invalid Examples
99 // identifier can’t be only digits
9abc // identifier can’t start with number
x+y // the only special character allowed is an underscore
for // it’s a reserved keyword
2my-first_name = "John" // W3Schools
my-first_name = "John"
