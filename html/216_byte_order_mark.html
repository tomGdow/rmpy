byte order mark
The byte order mark (BOM) is the Unicode code point U+FEFF
Can be used to indicate endianness of 16-bit and 32-bit endodings
Whether a text stream is Unicode
Which unicode encoding is being used
CAN interfere with utf-8 encoding
Examples
    "\N{BYTE ORDER MARK}"
    //    '\ufeff'

    "\N{BYTE ORDER MARK}".encode()
    //    b'\xef\xbb\xbf'
FEFF to utf-8
 
    (1) Third Byte 

        bin(int("FEFF",16))[-18:-12]

        //  '0b1111'

         -----------------
        | 1 1 1 0 x x x x |      1 1 1 0 1 1 1 1 
        -------------------

        hex(int("11101111",2))

        //    '0xef'

    (2) Second Byte 

        bin(int("FEFF",16))[-12:-6]

        //  '111011'

         -----------------
        | 1 0 x x x x x x |  1 0 1 1 1 0 1 1 
         -----------------

        hex(int("10111011",2))

        //   '0xbb'

    (3) First Byte 

        bin(int("FEFF",16))[-6:]

        //  '111111'

         -----------------
        | 1 0 x x x x x x |  1 0 1 1 1 1 1 1 
         -----------------

        hex(int("10111111",2))

        // '0xbf'
