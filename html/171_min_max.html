min()
Return the smallest item in an iterable or the smallest of two or more arguments
If one positional argument is provided, it should be an iterable.
If two or more positional arguments are provided, the smallest of the positional arguments is returned
There are two optional keyword-only arguments.
  1. The key argument specifies a one-argument ordering function like that used for list.sort()
  2. The default argument specifies an object to return if the provided iterable is empty. If the iterable is empty and default is not provided, a ValueError is raised
If multiple items are minimal, the function returns the first one encountered.
Remember capital letters will always be "less than" lower case
max()
Return the largest item in an iterable or the largest of two or more arguments
Syntax
min 

    min(iterable, *[, key, default])

    min(arg1, arg2, *args[, key])
max 

    max(iterable, *[, key, default])

    max(arg1, arg2, *args[, key])
Examples
Example 1 (Lists) 

    mylist=['tom', 'adam', 'Tomasdow']
    min(mylist)    //  'Tomasdow'
    max(mylist)    //  'tom'
    max(mylist, key=len)    //  'Tomasdow'
Example 2 (Dictionary) 

    mydict={'tom': 40, 'ziggy': 2, 'Tom':90}
    min(mydict)                        //   'Tom'
    min(mydict.values())               //   2
    min(mydict, key=lambda x: x[1])    //   'ziggy'
    min(mydict, key=mydict.get)        //   'ziggy'
    min(mydict.items(), key=lambda x: x[1])  //  ('ziggy', 2)
Example 3 (multiple args) 

    min(199, 2, 45, 4)
    //    2
Example 4 (Default value/Value Error) 

    min([])
    //  Traceback (most recent call last):
    //    File "<pyshell#28>", line 1, in <module>
    //      min([])
    //  ValueError: min() arg is an empty sequence

    min([], default='No Minimum')
    //    'No Minimum'
Example 5 (Single String) 

    min("thomas")    //   'a'
    min("tHomas")    //   'H'
