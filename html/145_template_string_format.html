template string
Template strings provide simpler string substitutions as described in PEP 292
Template strings support $-based substitutions, using the following rules:
  1. $$ is an escape; it is replaced with a single $.
  2. $identifier names a substitution placeholder matching a mapping key of "identifier".
  3. By default, "identifier" is restricted to any case-insensitive ASCII alphanumeric string (including underscores) that starts with an underscore or ASCII letter.
  4. The first non-identifier character after the $ character terminates this placeholder specification.

  5. ${identifier} is equivalent to $identifier.

    It is required when valid identifier characters follow the placeholder but are not part of the placeholder, such as "${noun}ification".

  6. Any other appearance of $ in the string will result in a ValueError being raised.
The string module provides a Template class that implements these rules.
The methods of Template

  1. substitute(mapping={}, /, **kwds)

    Performs the template substitution, returning a new string.

    mapping is any dictionary-like object with keys that match the placeholders in the template.

    Alternatively, you can provide keyword arguments, where the keywords are the placeholders. When both mapping and kwds are given and there are duplicates, the placeholders from kwds take precedence.

  2. safe_substitute(mapping={}, /, **kwds)

    Like substitute(), except that if placeholders are missing from mapping and kwds, instead of raising a KeyError exception, the original placeholder will appear in the resulting string intact.

    Also, unlike with substitute(), any other appearances of the $ will simply return $ instead of raising ValueError.

    While other exceptions may still occur, this method is called “safe” because it always tries to return a usable string instead of raising an exception.

    In another sense, safe_substitute() may be anything other than safe, since it will silently ignore malformed templates containing dangling delimiters, unmatched braces, or placeholders that are not valid Python identifiers.

template attribute
Template instances also provide one public data attribute
This is the object passed to the constructor’s template argument.
In general, you shouldn’t change it, but read-only access is not enforced.
 
    mytemplate.template           //  'Hello $name'

Examples
Example 1 
    from string import Template

    mytemplate = Template("Hello $name")
    student="Joe"
    mytemplate = Template("Hello $name")
    print(mytemplate.substitute(name=student))   //   Hello Joe
    mytemplate.template                          //  'Hello $name'
Example 2 
    from string import Template

    newtemp=Template("The shares of $company have $state. This is $reaction")
    newtemp.substitute(state='dropped', company="Apple", reaction="bad")
    //  'The shares of Apple have dropped. This is bad'

    newtemp.substitute(state='dropped', compaany="Apple", reaction="bad")
    //  Traceback (most recent call last):

Example 3 (David Beasley) 
    from string import Template

    s="$title"
    t=Template(s)
    t.substitute({'title': 'Hello World'})
    // 'Hello World'
    
    COMPARE

    s2='{title}'
    s2.format(title="Hello World")
    'Hello World'

    COMPARE 

    class Foo:
        def __init__(self):
            self.name = "Tom"

    f=Foo()
    "{x.name}".format(x=f)
    //  'Tom'
