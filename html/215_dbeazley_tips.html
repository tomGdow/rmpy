Comment on utf-8 encoding and memory
The internal representation of characters is almost never the same as how text is stored in files
Emphasis: text encodings (including utf-8 encoding refers to how text is stored in files not in computer memory (RAM)
Memory and files are two totally different representations of data
Internally, Unicode character codes are stored as arrays of C integers
Either 16 bit or 32 bit
To find out which:
    import sys
    sys.maxunicode
    //    1114111
    hex(sys.maxunicode)
    //    '0x10ffff'

    //    "An integer giving the value of the largest Unicode code point"
    //    D Beazley: value of 1114111 indicates 32-bit
                     value of 65535 indicates 16-bit
