pydoc — Documentation generator and online help system
xx
pydoc
The pydoc module automatically generates documentation from Python modules
The documentation can be presented as pages of text on the console, served to a Web browser, or saved to HTML files.
For modules, classes, functions and methods, the displayed documentation is derived from the docstring (i.e. the __doc__ attribute) of the object, and recursively of its documentable members.
If there is no docstring, pydoc tries to obtain a description from the block of comment lines just above the definition of the class, function or method in the source file, or at the top of the module
The built-in function help() invokes the online help system in the interactive interpreter, which uses pydoc to generate its documentation as text on the console
The same text documentation can also be viewed from outside the Python interpreter by running pydoc as a script at the operating system’s command prompt.
For example, run the following AS A SHELL prompt will display documentation similar in style to the manual pages shown by the Unix man command

    C:\Users\thoma> pydoc len
    C:\Users\thoma\AppData\Local\Programs\Python\Python39> pydoc test_module.py
The argument to pydoc can be the name of a function, module, or package, or a dotted reference to a class, method, or function within a module or module in a package.

Examples
Basic example 

    C:\Users\thoma> pydoc .\test_module_tgd.py

    Help on module test_module_tgd:

    NAME
        test_module_tgd - This module is a test module

    DESCRIPTION
        This module basically does nothing
        It may be used for testing out help and docstring functionality

    FUNCTIONS
        myfn()
            This function does nothing at all

    DATA
        mydict = {'dick': '2', 'harry': 3, 'tom': 1}
        mylist = [1, 2, 3, 4]
        mytuple = ('tom', 'dick', 'harry')

    FILE
        c:\users\thoma\test_module_tgd.py
Write to a plain text file 
    
    import pydoc
    import test_module_tgd
    f=open("test_module_tgd.txt", 'w')
    print(pydoc.render_doc(test_module_tgd, renderer=pydoc.plaintext), file=f, end='')
    f.close()

    // generates file 'test_module_tgd.txt', contents identical to that given 
    // in Example 1

    // NOTE 
    pydoc .\test_module_tgd.py > myfile.txt   // Just as good!!
                                                 // Identical to above
